So, lets begin with a simple Problem from a practise Exam. The problem says to define matrix-vector multiplication as a function in python, and check for reasonable errors. We assume that a vector is a list of intergers, and a matrix as a list of lists of integers, the vector representing in this case a list of column vectors, not rows. Now, we know from linear algebra, that a vector and a matrix can only be multiplied together, if the length of the column vector is the same as the width of the matrix. Otherwise, the length of the matrix is up to choice. Here is an outlook on the general first steps: (I do attempt to make reasonable formating choices)

def matrix_vektor_produkt(W:list[list[int]], x:list[int]) -> list[int]:
  if not isinstance(x, list):
    raise TypeError("Vector is not a list")
  if not isinstance(W, list):
    raise TypeError("Matrix not a list")
  for collumn in W:
    if not isinstance(collumn, list):
      rasie TypeError("Matrix not a list of lists")
  for collumn in W:
    if len(collumn) != len(W[0]):
      raise ValueError("Matrix not defined")
  if len(W) != len(x):
    raise ValueError("Not corresponding dimensions")
    
Now we can start with the first steps of the actual function. The function is supposed to return the corresponding result of the multiplication. The simplest way to compute the calculation is through the following. We compute the scalar product of each collumn with its corresponding vector member, and then add all the resulting collums together. We can write it out as follows: (important note, vector addition is defined is python only as concatenation of lists, not in the mathematical sense)

result = [0] * len(W[0])
for i in range(len(W)):
    for j in range(len(x)):
        result[i] += W[i][j] * x[j]
  return result

And our problem is finished. The complexity of this calculation is O(mn), where m and n are the matrix dimensions. This complexity however, is unavoidable, as every calculation is nececarry.
